このコマンドを受け取ったら、以下の手順で両アカウントの全ルームを分析する。

## 手順

### Step 1: ルーム一覧取得

**YutoKato（全室対象）**
```
mcp__chatwork-yuto__list_rooms()
```
→ 全ルームIDを取得（約79室、全件対象）

**くまお（フィルタリングあり）**
```
mcp__chatwork-kumao__list_rooms()
```
→ 以下のルールで絞り込む（最大150室）:
- **スキップ条件**（ルーム名に含む場合除外）: 「閉鎖」「旧」「通知」「SPOT」「LC通知」
- **優先対象**: DM（type=direct）、WF/リベクリ/就労/訪看 を含むグループ
- 上記で150室を超える場合は直近アクティブ順に上位150室

### Step 2: メッセージ取得

対象期間：**コマンド実行日の前日・前々日（過去2日分）**

```
mcp__chatwork-yuto__list_room_messages(room_id=xxx, force=1)
mcp__chatwork-kumao__list_room_messages(room_id=xxx, force=1)
```

- `force=1` で既読にかかわらず最新100件を取得
- タイムスタンプ（Unix秒）をJSTに変換し、**前日・前々日のメッセージのみ**を抽出
  - 例: 4月4日（金）に実行 → 4月2日（水）〜4月3日（木）が対象
  - 例: 月曜日に実行 → 土曜・日曜が対象（週末のログをカバー）
- 両アカウントのメッセージを統合

### Step 3: 分析

収集したメッセージを以下の観点で分析:

1. **統計**: 総メッセージ数、自分の発言数、返信受信数、アクティブルーム数
2. **TOP10ルーム**: メッセージ量が多い順
3. **完了タスク**: 「完了」「対応しました」「確認しました」等を含む発言
4. **進行中タスク**: 「対応中」「確認中」「検討中」等を含む発言
5. **意思決定事項**: 決定・承認・判断した内容
6. **持ち越し事項**: 「明日」「後日」「改めて」等で保留になった内容
7. **品質・クレーム**: 苦情、ミス、謝罪を含む発言
8. **改善提案**: 新しいアイデア、提案、要望

- [内容] （[ルーム名]）

### 🎯 意思決定事項（X件）
- [内容] （[ルーム名]）
...

### ⏭️ 持ち越し事項（X件）
- [内容] （[ルーム名]）

### ⚠️ 品質・クレーム（X件）

### Step 5: ダッシュボードに反映

レポート出力後、以下のコマンドで経営ダッシュボードのChatwork振り返りタブを更新する:

```bash
gh workflow run update.yml --repo ngu15c1027-cpu/wf-report
```

実行後2〜3分でGitHub Actionsが完了し、ダッシュボードの「💬 CW振り返り」タブに最新データが反映される。

> ※ ダッシュボードURL: https://ngu15c1027-cpu.github.io/wf-report/

---

## 注意事項

- 個人情報・機密情報は要約のみ表示（原文そのまま出力しない）
- 日付はJSTで判定（UTC+9）
- 対象期間は「実行日の前日」と「前々日」の2日分（当日は含まない）
- エラーが出たルームはスキップして続行
- くまおアカウントは室数が多いため、1ルームずつ順次処理する
